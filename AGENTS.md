# Global JFC Instructions

## Absolute Readability Rule

For all work for J.F. Carpio, forever: never use illegible gray text, low-contrast gray, faint decorative copy, opacity-based meaningful text, or any visual treatment that makes reading harder.

This applies to every project, page, report, deck, document, landing page, UI, prototype, and asset.

Text must be immediately readable. Hierarchy must come from size, weight, layout, spacing, borders, and deliberate accent color, not from making text faint.

Forbidden:
- Meaningful text in #999999, #AAAAAA, weak gray, faint muted styles, or opacity-reduced white.
- Low-contrast captions, notes, labels, helper text, form text, navigation, or metadata.
- "Premium subtle" gray that looks elegant but reads badly.

Required:
- Primary text on dark: #FFFFFF.
- Secondary text on dark: at least #F2F2F2.
- Tertiary/helper text on dark: at least #E6E6E6.
- Body copy and instructions must be readable on mobile and desktop without effort.

If an existing project has gray/low-contrast text, improve it when touching the file.

## Anti-Slop Visitor Copy Rule

Never expose internal instructions, prompt residue, implementation notes, editorial explanations, asset-selection rationale, or working-context clarifications in visitor-facing copy.

Forbidden visitor-facing copy includes phrases like:
- "shown first"
- "supporting views"
- "this image shows"
- "the version customized"
- "Cecilios is the tool"
- "example view"
- "actual demo" when used as an internal label rather than polished caption copy
- anything that explains why an asset was chosen instead of what the visitor should understand

Required:
- Captions must read as finished website copy.
- Visitor text must describe the actual client value, image content, or experience.
- If a clarification is needed for the assistant or project owner, keep it in comments, commit notes, or discussion, never in the rendered page.
